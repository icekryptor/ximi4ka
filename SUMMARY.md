## 🚀 Как запустить

### Вариант 1: Автоматический запуск (рекомендуется)

```bash
./start.sh
```

### Вариант 2: Вручную

**1. Подготовка PostgreSQL:**
```bash
# Установка
brew install postgresql@14
brew services start postgresql@14

# Создание БД
psql postgres -c "CREATE DATABASE ximfinance;"
psql postgres -c "CREATE USER ximfinance_user WITH PASSWORD 'ximfinance_pass';"
psql postgres -c "GRANT ALL PRIVILEGES ON DATABASE ximfinance TO ximfinance_user;"
```

**2. Инициализация данных (опционально):**
```bash
cd backend
npm run seed
```

**3. Запуск серверов:**

Терминал 1:
```bash
cd backend
npm run dev
```

Терминал 2:
```bash
cd frontend
npm run dev
```

**4. Открыть в браузере:**
```
http://localhost:5173
```

### Вариант 3: Docker

```bash
docker-compose up
```

## 🎯 Основные функции

### 1. Главная панель (Dashboard)
- Карточки статистики (доходы, расходы, баланс, количество)
- Последние 5 транзакций
- Период: последние 30 дней

### 2. Транзакции
- Таблица всех транзакций
- Создание новых транзакций (доходы/расходы)
- Редактирование и удаление

- Фильтр по типу (все/доходы/расходы)
- Сортировка по дате

### 3. Контрагенты
- Карточки контрагентов
- Типы: поставщик, клиент, оба
- Полная информация: ИНН, адрес, телефон, email
- Поиск по названию, ИНН, контактному лицу
- Статус активности

### 4. Категории
- Карточки категорий с цветами
- Раздельные категории для доходов и расходов
- Фильтр по типу
- Цветовая кодировка (8 цветов)
- Описания категорий

### 5. Отчёты и аналитика
- Финансовая сводка (доходы, расходы, баланс)
- График по категориям (Bar Chart)
- График по контрагентам (Pie Chart)
- Таблицы топ-10 по категориям и контрагентам
- Выбор периода

## 🐛 Решение проблем

### Backend не запускается
```bash
# Проверьте PostgreSQL
brew services list

# Если не запущен
brew services start postgresql@14

# Проверьте подключение
psql -U ximfinance_user -d ximfinance -h localhost
```

### Frontend не открывается
```bash
# Проверьте, что backend запущен
curl http://localhost:3001/health

# Проверьте порт 5173
lsof -i :5173
```

### Ошибка подключения к БД
- Проверьте `backend/.env`
- Убедитесь, что БД создана
- Проверьте пароль
