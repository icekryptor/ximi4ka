## Phase C — Recipe Engine

### TC-C-01: Создание `short_post` юнита

**Шаги:** `/content-bank` → «Создать юнит» (или эквивалентная кнопка). В content_type выбрать «📝 Короткий пост». Заполнить title=`Тест-пост про серную кислоту`. Сохранить.
**Ожидание:** Юнит создан. В списке отображается с badge «📝 Короткий пост».

### TC-C-02: RecipeView появляется для short_post

**Шаги:** Открыть только что созданный юнит в редакторе (двойной клик / кнопка edit).
**Ожидание:** В модалке ПОВЕРХ обычных полей появляется секция с заголовком «Короткий пост», описанием и кнопкой «Инициализировать рецепт» (recipe_state ещё пустой).

### TC-C-03: Initialize recipe

**Шаги:** Нажать «Инициализировать рецепт».
**Ожидание:** Toast «Рецепт инициализирован». Появляются 2 шага: «Драфт текста» (status=Ожидает, executor=AI) и «Финальный текст» (status=Ожидает, executor=Оператор).

### TC-C-04: AI-генерация драфта (тратит ~$0.05 Claude credits)

**Pre-condition:** ENV-4 ✓ (есть кредиты).
**Шаги:** На шаге «Драфт текста» нажать «Сгенерировать». Опционально написать в поле «Дополнительные инструкции для AI» что-то типа `пиши коротко, 300 символов`.
**Ожидание:** Через 5-30 секунд статус шага становится «На проверке», появляется StepEditor с textarea, в которой текст драфта (400-800 символов, без хэштегов).

### TC-C-05: Перегенерация драфта

**Шаги:** Не редактируя, нажать «Перегенерировать».
**Ожидание:** Новый текст в textarea (другой, чем был). Счётчик `Запусков: N` инкрементировался. **Важно:** старый текст из textarea пропал (потому что мы починили `key` prop на `StepEditor`).

### TC-C-06: Редактирование + Принять

**Шаги:** Отредактировать текст в textarea (например, добавить «🧪»). Нажать «Принять».
**Ожидание:** Статус шага → «Готово» (зелёный badge). Под шагом — read-only превью текста + кнопка «Открыть для правки».

### TC-C-07: «Открыть для правки» возвращает в `in_progress`

**Шаги:** На completed шаге нажать «Открыть для правки».
**Ожидание:** Шаг снова в редактируемом виде. Можно поправить и снова Принять.

### TC-C-08: Финальный шаг — Manual flow

**Шаги:** На шаге «Финальный текст» нажать «Начать шаг» (теперь должна появиться StepEditor — это был фикс плана).
**Ожидание:** Statys → «В работе», StepEditor с пустой textarea. Скопировать туда текст из draft (или ввести свой) → «Принять».

### TC-C-09: Concurrent AI generation — Set-based protection

**Pre-condition:** В одном юните инициализированы оба шага, draft в pending.
**Шаги:** В одном юните быстро дважды кликнуть «Сгенерировать» (между кликами <1s).
**Ожидание:** Второй клик игнорируется (или disabled). Только одна Claude-генерация выполняется. Никакого «дубль-spend» кредитов.

### TC-C-10: AI-шаг при отсутствии кредитов — graceful error

**Pre-condition:** (опционально / симулируется) счёт Anthropic в нуле или token невалидный.
**Шаги:** Нажать «Сгенерировать».
**Ожидание:** Красный toast: «Закончились кредиты Anthropic — пополни баланс в console.anthropic.com → Billing» (или аналогичное). НЕ generic «Ошибка генерации».

### TC-C-11: Не-short_post тип НЕ показывает RecipeView

**Шаги:** Открыть юнит с content_type=`short_video` или `text_post`.
**Ожидание:** RecipeView НЕ отображается. Legacy production-блок (script_text, video_brief и т.д.) — как раньше.

## Phase D — Telegram Auto-Publishing

**Все TC-D-* требуют ENV-3, ENV-6 (бот + канал).**

### TC-D-01: Настройка Telegram-канала в системе

**Шаги:**
1. `/settings/channels` → создать или отредактировать Telegram-канал.
2. slug=`telegram_main` (или любое), display_name=`Тестовый ТГ`, platform=`telegram`.
3. В появившемся блоке «Telegram chat ID» ввести numeric ID канала (получить через @userinfobot, обычно `-100…`).
4. integration_status=`api_connected`, active=✓. Сохранить.

**Ожидание:** Канал сохранён. В DB `config_json.chat_id` = тот ID.

### TC-D-02: PublicationsEditor — checkbox enabled для api_connected

**Шаги:** Открыть юнит short_post (с completed final step). В PublicationsEditor добавить публикацию для канала `telegram_main`. Должно появиться поле «scheduled_at» + чекбокс «Авто-публикация».
**Ожидание:** Чекбокс **enabled** (canAuto=true). Tooltip отсутствует.

### TC-D-03: Checkbox disabled для manual канала

**Шаги:** В том же юните добавить публикацию для канала с `integration_status=manual` (например `tiktok`).
**Ожидание:** Чекбокс **disabled**. Hover → tooltip: «Канал не настроен для авто-публикации (нужен api_connected)».

### TC-D-04: Авто-публикация в Telegram — happy path

**Шаги:**
1. Сохранить юнит short_post с completed final.
2. В Telegram-публикации задать `scheduled_at` = «через 2 минуты», поставить «Авто-публикация».
3. Закрыть модалку. Подождать 2-3 минуты.

**Ожидание:**
- В Telegram-канале появилось сообщение с текстом final-артефакта.
- В UI на этой публикации появилось значение `published_at` и `published_url` (ссылка вида `https://t.me/c/…/N`).
- DB `content_publications.publisher_log` содержит `{ success: true, attempts: 1, completed_at, raw: {...} }`.

### TC-D-05: Manual publish-now

**Шаги:** Создать ещё одну Telegram-публикацию для другого юнита (или удалить старую `published_at` и попробовать). Через прямой API:
```bash
curl -X POST https://<backend>/api/content-publications/<id>/publish-now \
  -H "Authorization: Bearer <jwt>"
```
**Ожидание:** Возвращает 200 с обновлённым публикацией. Сообщение в Telegram. `publisher_log.manual: true`.

### TC-D-06: Double-click defense

**Шаги:** На уже published-публикации (где есть published_at) попробовать ещё раз publish-now.
**Ожидание:** HTTP 409, тело `{ error: 'Публикация уже опубликована' }`. **Второе сообщение в Telegram НЕ отправлено.**

### TC-D-07: Mid-flight recipe — публикация падает с понятной ошибкой

**Шаги:** Создать short_post юнит, инициализировать рецепт, **но НЕ дойти до final.completed**. Назначить Telegram-публикацию с `auto_publish=true` через 1 минуту.
**Ожидание:**
- Сообщение в Telegram **НЕ отправлено**.
- В `publisher_log.last_error` появляется текст «Рецепт не завершён: шаг "final" не имеет готового текста. Завершите шаг или отключите авто-публикацию.»
- После 3 попыток `auto_publish=false`, `gave_up=true`.
- В PublicationsEditor — красный badge «❌ Авто-публикация остановлена», hover показывает last_error.

### TC-D-08: Telegram-channel URL для `@username` — корректный

**Шаги:** Создать канал с chat_id=`@публичный_тестовый_канал_username`, integration_status=api_connected. Опубликовать.
**Ожидание:** `published_url` = `https://t.me/публичный_тестовый_канал_username/<msg_id>` (без `/c/`). Ссылка открывается в браузере.

### TC-D-09: Длинный текст (>4000 chars) — truncation работает

**Шаги:** В final-артефакте написать ~5000 символов. Запустить auto-publish.
**Ожидание:** В Telegram приходит сообщение из ~3900 символов + `…` в конце. НЕТ ошибки «Bad Request» от Telegram (значит HTML-сущности не разорваны).

## Cross-cutting / Edge cases

### TC-X-01: Publisher_log badge — gave_up

**Ожидание:** **Жёлтый** badge «⚠ Попыток: 1». После исправления и успешной публикации badge пропадает.

### TC-X-03: Rebuild на Railway копирует YAML рецепты

**Ожидание:** Строка присутствует. Если её НЕТ — Critical-блокер из Phase C-review всё-таки прошёл в прод; немедленно тикет на постмортем.

### TC-X-04: Регресс по legacy content_type
